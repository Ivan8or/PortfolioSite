    Intro

    This project is more technical than most, as it is actually a software library I made for myself to make it possible for my minecraft servers to talk to each other, letting me do cool things like have players generate brand new worlds just by clicking a button.

    While the minecraft Bukkit API does have some support for intra server communication via plugin messaging, their solution is not free of shortcomings. Most relevant to my situation is that their messaging will not work for a server that has no players online - unacceptable for me!

    As a result, I decided to set out and instead implement my own messaging system.

    Image by brgfx on Freepik

    Goals

    Most message broker solutions are a middleware, where the user needs to host a broker service which then manages communications sent through it. This allows messages to be queued if the recipient is unreachable, as well as to multicast messages efficiently.

    For my usecase, I did not need this - I wanted something different; something simpler. A system which would be ready to use without any additional setup, tailored to hobby developers who simply want a reliable way for two programs to communicate.
